конфиг - для токена и констант 
основной файл "бот" - для шаблона, описания событий при нажатии кнопок etc.

прототип бота будет работать по принципу лонг-пуллинг (как и любой человеческий бот),
ожидая нажатия кнопки /start; 
после старта выскакивает приветствие, описание сервиса.

если будет использоваться лишь один алгоритм шифрования, то все будет куда проще в будущем с БД и прочим.

если прототип окажется рабочим и безопасность шифра удволетворяющей (меня), можно будет переделывать сервис под хранилище с шифратором, работа с SQL на тебе

для твоего удобства пишу на pytelegramBotAPI, так что связать БД с ботом не будет сложным
